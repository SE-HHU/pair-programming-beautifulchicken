### 概要部分

| 代码能符合需求和规格说明么？   | 符合需求与规格说明                                                        |
| ---------------- | ---------------------------------------------------------------- |
| 代码设计是否有周全的考虑？    | 代码设计周全性较高，虽然使用传统的面向过程编程，但是这样的优点是稳定性高、速度快。                        |
| 代码可读性如何          | 目前来讲由于我们的项目更贴近工业级软件的代码，因此可读性不如面向用户等更加落地的项目的代码好，这也是我们后续需要进行改进的地方。 |
| 代码容易维护吗？         | 由于封装性较低耦合度较高，因此可维护性没有那么高。                                        |
| 代码的每一行都执行并检查过了吗？ | 所有的代码，目前经过我们的测试，没有发现问题，但是依旧可能存在一些我们没有发现的问题，也希望大家能够积极提交issue～     |
